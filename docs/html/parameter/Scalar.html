Scalar parameter.

A scalar parameter is a parameter for which its value is a scalar (numpy scalar). It accepts the following contraints:

  • Particular dtype (e.g., to restrict the scalar to an integer or float).

  • Lower bound (minimum value below which the scalar cannot be).

  • Upper bound (maximum value above which the scalar cannot be).

__init__(dtype=None, min=None, max=None, strict=False)
Parameters

  • dtype (np.dtype, optional) – Dtype of the parameter. The default accepts any dtype.

  • min (scalar, optional) – Lower bound of the parameter (minimum accepted value). The default assumes no lower bound.

  • max (scalar, optional) – Upper bound of the parameter (maximum accepted value). The default assumes no upper bound.

  • strict (bool, optional) – If True, the value must be in the open interval (min, max). Otherwise, it can be in the closed interval [min, max]. The default is False (closed interval).

Raises

ValueError – If min and max are not scalars. If max is not greater than min.

control(value=None, name='value')[source]

Control whether the value of the parameter is valid or not.

The value of the parameter is valid if it satisfies all the following conditions:

  • Scalar (ndim equal to 0).

  • Correct dtype.

  • Greater than the lower bound.

  • Less than the upper bound.

Parameters

  • value (array-like or None, optional) – The value to control. If None, the value of the instance will be used instead. The default is None.

  • name (str, optional) – Name of the parameter. If an error is raised, this name will be used. The default is “value”.

Raises

ValueError – If the parameter is not initialized (set to None). If one of the condition above is not fulfilled.
