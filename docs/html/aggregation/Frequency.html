Number of values in each group. The first element is the number of values in the first group, the second element is the number of values in the second group, … The length corresponds to the total number of groups.

aggregate(func='sum', observed=False, to_frame=False, **kwargs)[source]

Aggregate the values by group.

Parameters

  • func (string or callable, optional) – Function to use to aggregate values belonging to a same group. It can be a callable or a string such as “sum”, “mean”, “std”, “var”, “min”, “max” or “median”. The default aggregation is a sum.

  • observed (bool, optional) – If True, only groups where at least one value is observed are shown in the output. Otherwise, groups where no value is observed are also shown in the output. The default is False.

  • to_frame (bool, optional) – If True, a dataframe is returned where each row of the dataframe corresponds to an aggregated group. Otherwise, an array is returned. The default is False.

  • **kwargs – Additional keyword arguments for the func function when a callable is passed.

Returns

Aggregated values by group.

Return type

array or dataframe

split()[source]

Splits the values by group.

The values attribute is split into multiple sub-arrays. The number of sub-arrays is equal to the total number of groups i.e. the length of the frequencies attribute. The i-th sub-array corresponds to all individual values associated with the i-th group.

Returns

List of sub-arrays splitting the values attribute by group. The length of the list is equal to the length of the frequencies attribute.

list of array

classmethod random(frequency, severity, size=None)[source]

Draw random samples of a frequency-severity model.

The random samples are generated as follows:

  • Draw \(n\) independent random samples from the frequency distribution given by the frequency argument where \(n\) is an integer given by the size argument (number of scenarios/groups to be drawn). This leads to a vector

    \[(N_1, ..., N_n)\]

    where \(N_i\) is the number of claims for the i-th scenario.

  • Draw \(m\) independent random samples from the severity distribution given by the severity argument where \(m\) is the total number of individual claims for all scenarios equal to \(\sum_{i=1}^n N_i\). This leads to a vector

    \[(X_{1,1}, X_{1,2}, ..., X_{1,N_1}, ..., X_{n,1}, X_{n,2}, ..., X_{n,N_n})\]

    where \(X_{i,j}\) is the cost of the j-th individual claim associated with the i-th scenario.

Note that all random samples are generated independently of each other and that the random number generator used is the one associated with the frequency and severity distribution.

Parameters

  • frequency (UnivariateDistribution or 1D array-like) – Frequency distribution. The number of claims will be randomly generated from this distribution. If the number of claims don’t need to be randomly generated (already known), a 1D array can be provided instead.

  • severity (UnivariateDistribution) – Severity distribution. The cost of each individual claim will be randomly generated from this distribution.

  • size (int or None, optional) – Number of scenarios to randomly generate. This argument is only useful if the frequency argument is an univariate distribution and not an array. The default is None.

Raises

  • TypeError – If frequency is not an univariate distribution or a 1D array. If severity is not an univariate distribution.

  • ValueError – If the generated number of claims from frequency doesn’t result in a 1D array of non-negative integers.

Returns

Frequency-severity model.

Return type

Frequency
