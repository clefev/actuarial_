Mixture

class actuarial.superclass.Mixture[source]

Bases: object

Superclass for building mixture models.

A mixture model is described by a collection of weighted components. The individual components that are combined to form the mixture model are called the mixture components and the weights associated with each component are called the mixture weights.

A mixture distribution is an example of mixture model. The mixture components are the individual distributions \(X_k\) and the mixture weights are the probabilities \(w_k \in (0,1)\) associated with each distribution.

__init__(components, weights=None, component_type=None, relative_weights=False)[source]
Parameters

  • components (list) – List containing the mixture components.

  • weights (list of array-like or None, optional) – Weights associated with the mixture components. The list must have the same length as the number of components. The default assumes a weight of 1 associated with each component.

  • component_type (type or None, optional) – Type of the mixture components. Each component should have this type. If None, components can have any type. The default is None.

  • relative_weights (bool, optional) – If True, the weights are considered as probabilities and are normalized to get a sum equal to 1. The default is False.

Raises

  • TypeError – If components is not a list or tuple. If the elements of components don’t have the correct type given by component_type.

  • ValueError – If the length of weights is different from the length of components. If the elements of components and weights cannot be broadcast together. If negative weights are provided when relative_weights is set to True.
