actuarial.util.aggregate_by(x, by, func='sum', sort=False, axis=0, kwargs={})[source]

Aggregate data by groups along an axis. The data belonging to a same group will be aggregated together using the func function.

Parameters

  • x (array-like) – Input array containing the data to aggregate by groups.

  • by (1D array-like or pd.Categorical) – Grouping elements along the aggregation axis of the input array. Therefore, the length must correspond to the size of x along the aggregation axis. Data belonging to a same group will be aggregated together.

  • func (string or callable, optional) – Function to use to aggregate data belonging to a same group. It can be a callable or a string such as “sum”, “mean”, “std”, “var”, “min”, “max”, “median”, “prod” or “count”. The default aggregation is a sum.

  • sort (bool, optional) – If True, the output groups will be sorted in ascending order. Otherwise, they are given in the order they appear in the by argument. The default is False.

  • axis (int, optional) – Axis along which the data are aggregated. The default is the first axis of the input array x.

  • kwargs (dict, optional) – Additional keyword arguments for the func function when a callable is passed.

Returns

  • groups (array) – Output groups after aggregation. The groups are sorted in ascending order if the sort argument is True. The returned array is along the aggregation axis and can be broadcast with the aggregate output.

  • aggregate (array) – Aggregated values by groups. The shape is the same as the input array x except that the size along the aggregation axis is equal to the number of output groups len(groups).

Raises

ValueError – If the length of by doesn’t correspond to the size of x along the axis.

actuarial.util.is_subshape(shape1, shape2)[source]

Indicates wether a shape is a subshape of another shape.

Note

Shape1 is a subshape of shape2 if its number of dimensions (length) is less than or equal to the number of dimensions of shape2 and if the size along each axis of shape1 is equal to 1 or the size of shape2 along the corresponding axis. Equivalently, shape1 is a subshape of shape2 if they can be broadcast together to a single shape and if the resulting shape is the same as shape2. Finally, note that shape1 is a subshape of shape2 is equivalent to shape2 is a supershape of shape1.

Parameters

  • shape1 (tuple of int) – The shape to control if it is a subshape.

  • shape2 (tuple of int) – The shape against which shape1 is tested as a subshape.

Returns

True if shape1 is a subshape of shape2 and False otherwise.

Return type

bool

actuarial.util.lower_bound(x, table, strict=False, index=False)[source]

Given a vector of non-decreasing breakpoints in table, find the lower bound of each element of x in table. The lower bound index of the value x in table is the index i satisfying table[i] <= x < table[i+1]. If the value x is below min(table), its index will be truncated to 0. The function returns the index i if the index argument is True and the value table[i] if it is False.

Parameters

  • x (array_like) – Values for which the lower bound must be found in table.

  • table (1D array_like) – Must be sorted in ascending order.

  • strict (bool, optional) – If True, the interval is open at left and closed at right: table[i] < x <= table[i+1]. The default is False.

  • index (bool, optional) – If True, returns the indices of the lower bound instead of the values. In this case, the output array will contain integers between 0 and len(table)-1. The default is False.

Returns

Array of lower bound values with the same shape as x.

Return type

array

Normalize an axis in order to have a positive value such that 0 <= normalized_axis < ndim.

Parameters

  • axis (int or iterable of int or None) – Input axis. A single axis can be provided through an integer and multiple axis can be provided through an iterable of integers.

  • ndim (int) – Number of array dimensions.

  • negative (bool, optional) – If True, the normalized axis will be negative such that -ndim <= normalized_axis < 0 (otherwise, it will be positive such that 0 <= normalized_axis < ndim). The default is False.

  • sort (bool, optional) – If True and if multiple axis are provided, the normalized axis will be sorted in ascending order. The default is False.

  • allow_duplicate (bool, optional) – If False and if multiple axis are provided, disallow an axis from being specified twice. The default is False.

Normalized axis. The output can be an integer, a tuple of integers or a None value following the type of the input axis (respectively, integer, iterable of integers or None).

Return type

int or tuple of int or None

Raises

  • AxisError – If any axis provided is out of range (i.e. if we don’t have the following inequality: -ndim <= axis < ndim).

  • ValueError – If an axis is repeated multiple times (unless allow_duplicate is set to True).
