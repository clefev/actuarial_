Parameter

The parameter module provides all the tools needed to work with parameters.

Module description

The parameter module can be easily imported from the actuarial package:

from actuarial import parameter

Several classes are implemented in this module.

First, we can find an abstract class called Parameter. This abstract class allows building parameter classes in a standardized way and with various features pre-implemented. Therefore, all parameter classes implemented in this package inherit from this abstract class. Moreover, it also allows you to build and customize your own parameter classes (by inheriting from this abstract class). Related documentation can be found below:

class actuarial.parameter.Parameter[source]

Bases: abc.ABC, actuarial.superclass.Arithmetic.Arithmetic, actuarial.superclass.Base.Base, actuarial.superclass.Broadcast.Broadcast, actuarial.superclass.Comparison.Comparison

Abstract class for building parameters.

Parameters are used by parametric models. There are different types of parameters such as scalar parameters, array parameters, correlation matrix parameters, … Each type of parameter should be represented by a dedicated class inheriting from this abstract class. The parameters can also have additional constraints. For example, a scalar parameter could have a range of accepted values i.e. a lower bound and upper bound. These constraints should be set in the __init__ method of the dedicated class.

Once a parameter instance is initialized, no value is associated with the parameter. A value can be associated with the parameter by using the set method. This value will then be accessible through the value attribute. In order to control that this value is valid, the control method can be used.

abstract control(value=None, name='value')[source]

Abstract method to control whether the value of the parameter is valid or not. If it is not valid, a ValueError should be raised.

The first argument must be the value to control. By convention (and by default), if the value to control is set to None, the value of the instance will be controlled instead. The second argument must be the name of the parameter. If an error is raised, this name should be used. Other additional arguments are not allowed.

Secondly, the module offers different parameter classes. For example, it implements scalar parameters, array parameters, correlation matrix parameters, … All these parameter classes inherit from the Parameter abstract class as described above. In this way, they all share a common set of features (properties and methods) and they can be easily identified as a parameter (using Python’s built-in functions isinstance or issubclass). The following parameter classes are implemented in this module:

  • Scalar

  • Array

  • Correlation matrix

Finally, the parameter module provides a Parameters class (with a “s” at the end). This class represents a collection of parameters. It is used to efficiently store and manage multiple parameters. In particular, it allows easy storage, extraction and controls of several parameters. The documentation of this class can be found below:

class actuarial.parameter.Parameters[source]

Bases: actuarial.superclass.Dictionary.Dictionary, actuarial.superclass.Base.Base, actuarial.superclass.Broadcast.Broadcast

Class representing a collection of parameters.

This class inherits from the Dictionary superclass. Therefore, it represents a collection of key-parameter pairs. The key is a string representing the name of the parameter and the parameter is a Parameter instance. This class allows to control, update and reset several parameters at once.

control()[source]

Control whether the value of each parameter is valid or not.

For each parameter, the associated control method will be called. If a parameter is not valid, an error will be raised. If the instance contains a parameter called min and a parameter called max, an additional control is performed to ensure that min < max.

Raises

ValueError – If a parameter is not valid. If the instance contains a parameter called min and a parameter called max for which min >= max.
